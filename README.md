# CS230
Briefly summarize The Gaming Room client and their software requirements. Who was the client? What type of software did they want you to design?
The client, The Gaming Room, is looking to have their existing mobile app Draw It or Lose It developed into a web-based app that can be run on various platforms. The ideal solution for the client involves using a coding framework that allows us to create applications for multiple platforms. This multi-platform coding framework may lead to some inconsistencies between platforms (web-based vs. Android) and limited access to platform-specific features in app. 

What did you do particularly well in developing this documentation?
I think my research into various operating systems was well done because it provides the client with the benefits and drawbacks between the platforms available to develop their project.

What about the process of working through a design document did you find helpful when developing the code?
The process of creating a UML diagram help to visualize the objects that the application will consist of and their relationships to other ojects.

If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?
If I had to revise one component of my document it would the recommmendations section because it could've been more extensive.

How did you interpret the user’s needs and implement them into your software design? Why is it so important to consider the user’s needs when designing?
Organizational documents were analyzed to determine functional/nonfuncational requirements, design constraints, and user needs. It's important to consider user needs because those needs can be translated into required system functionality. 

In order to deliver my software design to the client, I initially began by gathering functional and nonfunctional system requirements. Design constraints were later determined and the collected information was used to illustrate the system architecture. UML diagrams provided a blueprint of the proposed system, and operating platform analysis was conducted to present the client with the best recommendation for the ideal environment to develop the application.
